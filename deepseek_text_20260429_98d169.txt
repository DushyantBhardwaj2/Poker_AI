## 7. Database Design

### MongoDB Collections

#### Users
| Field | Type | Description |
|-------|------|-------------|
| _id | ObjectId | Auto-generated |
| email | String | Unique login |
| passwordHash | String | Bcrypt hashed |
| createdAt | Date | Account creation date |

#### Games
| Field | Type | Description |
|-------|------|-------------|
| _id | ObjectId | Unique game ID |
| userId | ObjectId | Owner (the assistant user) |
| tableSize | Number | Maximum seats |
| blinds | Object | `{ small: Number, big: Number, ante?: Number }` |
| players | Array of `Player` subdocs | Seated players with position, stack |
| communityCards | Array of Strings | Cards dealt (e.g., ["Ah","Kd"]) |
| pot | Number | Main pot |
| sidePots | Array | `[{amount, eligiblePlayers}]` |
| round | String | preflop/flop/turn/river/showdown |
| dealerIndex | Number | Current dealer position |
| currentPlayerIndex | Number | Who must act |
| actions | Array of `Action` subdocs | Entire action history |
| status | String | waiting/active/finished |
| result | Mixed | Final winner(s), hand ranks |

#### Actions (embedded in Games)
| Field | Type | Description |
|-------|------|-------------|
| playerIndex | Number | Seat index |
| type | String | fold/check/call/raise/all-in |
| amount | Number | Amount (0 for fold/check) |
| timestamp | Date | When action occurred |

#### HandHistory
| Field | Type | Description |
|-------|------|-------------|
| _id | ObjectId | Auto |
| gameId | ObjectId | Reference to original game |
| hands | Array | Each player's hole cards (if shown) |
| communityCards | Array | Final board |
| actions | Array | Full action log |
| winner | Mixed | Winner info & net profit |
| metadata | Object | Date, duration, etc. |

**Relationships:**  
- One User has many Games.  
- One Game embeds Players and Actions for performance.  
- HandHistory is a separate collection for completed hands to enable fast analytics.

## 8. API Documentation

### Base URL: `https://api.pokersense.ai/v1` (replace with localhost:5000/api for development)

| Endpoint | Method | Purpose | Request Body | Response |
|----------|--------|---------|--------------|----------|
| `/game/create` | POST | Create a new poker game | `{ "tableSize": 9, "blinds": { "small": 1, "big": 2 } }` | `{ "gameId": "...", "status": "waiting" }` |
| `/player/add` | POST | Add a player to a game | `{ "gameId": "...", "name": "Tom", "stack": 200, "seatIndex": 0 }` | `{ "success": true, "playerId": "..." }` |
| `/cards/set` | POST | Set hole/community cards | `{ "gameId": "...", "type": "hole"|"community", "playerIndex": 0, "cards": ["As","Ks"] }` | `{ "gameState": {...} }` |
| `/action/add` | POST | Record a player action | `{ "gameId": "...", "playerIndex": 0, "action": "raise", "amount": 12 }` | `{ "pot": 34, "nextPlayer": 2, "valid": true }` |
| `/game/state` | GET | Get current game state | `?gameId=...` | Full game object |
| `/ai/suggest` | POST | Get AI recommendation | `{ "gameId": "..." }` | `{ "move": "raise", "amount": 18, "confidence": 0.82, "reasoning": "..." }` |
| `/ai/bluff` | POST | Predict bluff probability per opponent | `{ "gameId": "...", "targetPlayerIndex": 1 }` | `{ "bluffProbability": 0.65, "type": "loose-aggressive" }` |
| `/ai/opponent` | GET | Get opponent profile | `?gameId=...&playerIndex=1` | `{ "type": "TAG", "vpip": 22, "pfr": 18, "af": 3.2 }` |
| `/hand-history` | GET | List completed hands | `?userId=...` | Array of hand summaries |
| `/hand-history/:id` | GET | Get full hand replay | - | Detailed actions & results |

**Authentication** (after Phase 2):  
Include `Authorization: Bearer <token>` header for all endpoints.
